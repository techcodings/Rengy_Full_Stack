1. Project Overview

TeamVault is a production-ready, full-stack web application built using the MERN stack (MongoDB, Express.js, React, Node.js) that implements a team-scoped Role-Based Access Control (RBAC) system.

The application enables organizations to manage users, teams, roles, and permissions with a dynamic authorization model where a user can hold different roles in different teams.

Key Features

  • User Management — Create users with unique email, search and filter with pagination
  • Team Management — Create teams, add/remove members, view team compositions
  • Role Management — Create reusable roles (Admin, Manager, Viewer) with configurable permissions
  • Permission Collection — Permissions stored as first-class DB entities (not hardcoded strings)
  • Team-Scoped RBAC — User A can be Admin in Team Alpha but Viewer in Team Beta
  • Permission Resolver — Dynamic UI to query any user's effective permissions in any team
  • JWT Authentication — Secure login/register with token-based auth
  • Authorization Middleware — Reusable checkPermission('CREATE_TASK') middleware

4. Database Design

The system uses 5 MongoDB collections with proper ObjectId references:

CollectionFieldsPurpose
usersname, email (unique), password (hashed)User accounts
teamsname, description, createdByTeam entities
permissionsname (unique, UPPER_SNAKE_CASE), descriptionPermission definitions
rolesname (unique), permissions: [ObjectId→Permission], descriptionRoles with permission refs
membershipsuserId→User, teamId→Team, roleId→RoleUser-Team-Role mapping
Critical Design Decision: Roles are NOT stored on the User document. The Membership collection maps (userId, teamId, roleId), enabling a user to hold different roles in different teams. This is the industry-standard approach for team-scoped RBAC.

Relationship Diagram

  User ──────┐
              │  userId
              ▼
         Membership ◄── Compound Unique Index (userId + teamId)
              │
   teamId ────┤──── roleId
              │         │
              ▼         ▼
            Team      Role ──→ permissions: [ObjectId]
                                     │
                                     ▼
                                Permission { name, description }

5. RBAC Permission Flow

When a protected API request is made, the authorization follows this flow:

1. JWT Token→ 2. auth.js→ 3. checkPermission()→ 4. Membership Lookup→ 5. Role → Permissions→ 6. Allow/Deny
  1. Client sends request with Authorization: Bearer <JWT> + teamId
  2. auth.js middleware verifies JWT → attaches req.user
  3. checkPermission("CREATE_TASK") middleware runs
  4. Looks up Membership for (userId, teamId)
  5. Deep populates: Role → Permission[] → extracts permission names
  6. If permission found → next() | If not → 403 Forbidden

Usage Example

// In route definition:
router.post('/tasks',
  auth,                             // Step 1-2: Verify JWT
  checkPermission('CREATE_TASK'),   // Step 3-6: Check permission
  taskController.createTask         // Handler runs only if authorized
);

9. Setup & Installation Guide

Prerequisites

  • Node.js v18 or higher — nodejs.org
  • MongoDB running locally on port 27017 — mongodb.com
  • Git installed — git-scm.com

Step 1: Clone the Repository

git clone https://github.com/techcodings/Rengy_Full_Stack.git
cd Rengy_Full_Stack

Step 2: Setup Backend

cd server
npm install

Step 3: Configure Environment Variables

# Copy the example env file
cp .env.example .env

# The .env file should contain:
PORT=5000
MONGODB_URI=mongodb://localhost:27017/teamvault
JWT_SECRET=your_jwt_secret_key_here
JWT_EXPIRES_IN=7d
NODE_ENV=development

Step 4: Start MongoDB

# Make sure MongoDB is running on localhost:27017
# On Windows: net start MongoDB
# On Mac: brew services start mongodb-community
# On Linux: sudo systemctl start mongod

Step 5: Seed the Database

npm run seed

This creates: 4 permissions, 3 roles, 5 users, 3 teams, and 9 memberships.

Step 6: Start the Backend Server

npm run dev

Server runs at http://localhost:5000

Step 7: Setup Frontend (New Terminal)

cd ../client
npm install
npm run dev

Frontend runs at http://localhost:3000

Step 8: Open the Application

# Open in browser:
http://localhost:3000

# Login with seed credentials:
Email:    alice@example.com
Password: password123
Quick Start (All Commands):

cd server && npm install && cp .env.example .env && npm run seed && npm run dev
